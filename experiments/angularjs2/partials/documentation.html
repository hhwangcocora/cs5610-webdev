Documentation

In this experiment, I practised using Angular JS Routing mechanism to navigate between pages without reloading them.

It configures $routeProvider to route for different pages in a a centralized place, then use ng-view to embed the page which is navigated to. Since the page is not reloaded entirely, the state of the which page is currently loaded is stored in url. Thus user can click back and forth history buttons.

Also I put a path-like navigation when it is on sub pages. For example Home/Documentation. This allows user to understand the website hierarchy.
